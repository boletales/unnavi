## tl;dr
岡田悠氏の記事「[目的地のわからないナビアプリを作った](https://note.com/hyosasa/n/n917c1b0efb06)」に触発され、ランダムな目的地への距離だけ表示する[アプリケーション](https://boletales.github.io/unnavi/)を作成した。歩いてたどり着けそうな場所だけを目的地として選定するため、OpenStreetMap のデータを利用して路地のみを選択するようにしている。

## OpenStreetMap について
[OpenStreetMap（OSM）](https://www.openstreetmap.org/)は、ユーザーが自由に調査編集できる地理情報データベースである。ウィキペディアがフリーの百科事典であるように、OSMはちょうどフリーの地図であると思ってよい。上記サイトを開いて表示される地図は、全世界のユーザー（マッパーと呼ばれる）が追記した地理情報を地図画像の形に描画したものである。
OSMの本体は、地物の位置を示した「ノード」、ノードをつなげた「ウェイ」、連関を表す「リレーション」と、これらの属性を示す「タグ」から構成されるベクトルデータである。たとえば道路であったら、線形に沿って小刻みにウェイを配置して、それぞれに状態を表すタグ（路線名を表す `name=国道418号線`、国道であることを表す `highway=trunk`、車線数を表す `lanes=1` などさまざまある）を付与し、これらのウェイに対して路線全体のリレーションを定義する、といった具合である。
ユーザーの力でオープンな地図を作るという試みは傍目には無謀そうに映るが、全世界のマッパーの尽力と、各種機関の協力（例えば、Bing Mapsの航空写真データや国土地理院地図のデータはトレース元としての利用が許可されている）に支えられ、2026年現在、日本のOSMの道路データは実用に足る品質と言ってよいだろう。
少なくとも、散歩のためにわざわざGoogle Maps APIに金を払いたくない私のような開発者にとって、[OpenStreetMap](https://www.openstreetmap.org/copyright)への帰属表示さえすれば自由に利用できるOSMは極めてありがたい存在である。

## Overpass API について
OSMの生データを直接利用するのは骨が折れる。今回のように「路地を取得したい」などという場合には、OSMのデータをタグ種別・地域などで絞り込んで取得できるOverpass APIというものを使うとよい。[overpass-api.de](https://overpass-api.de/)がOverpass APIのインスタンスを利用制限なく無料で提供してくれているので、ありがたく使わせていただいた。

## 実装の概略
現在地から目的地までの大まかな距離はユーザーに指定させることとした。あらかじめクライアント側で候補点をひとつ選んだのち、Overpass APIを用いて、候補点から100m圏内の路地を取得する。具体的には、次のようなクエリを渡している：
```
[out:json][timeout:25];
way["highway"~"^(residential|living_street|pedestrian|footway|path|unclassified)$"]["access"!~"^(private|no|customers)$"]["indoor"!="yes"](around:100,35.000,135.000);out geom;
```
このクエリでは、ウェイのうち道路（`highway`タグを持つ）で、路地や歩道に該当し、私有地や立ち入り禁止でなく屋内でもないものを、(35.000,135.000)を中心に半径100mの範囲で取得している。クエリの構文については、[Overpass QLのドキュメント](https://wiki.openstreetmap.org/wiki/JA:Overpass_API/Overpass_QL)を参照されたい。試しにクエリを作って遊んでみたい、という場合は[Overpass Turbo](https://overpass-turbo.eu/)というWebアプリケーションを使うとクエリの構築や結果の表示を手軽に行える。
最後に、このようにして取得した路地の中からランダムにひとつ選び、最終的な目的地として距離を表示している。
現在は歩道と細い道を対象としているが、公園やらコンビニやらを目的地として選んでみても面白そうだ。
